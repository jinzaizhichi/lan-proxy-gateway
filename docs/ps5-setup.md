# PS5 网络设置指南

在电脑上启动 LAN Proxy Gateway 后，按以下步骤配置 PS5。

## 前提条件

- 电脑已运行 `sudo gateway start` 且显示启动成功
- 记下电脑的局域网 IP（启动时终端会显示，例如 `192.168.1.2`）
- PS5 和电脑连接同一个 Wi-Fi / 路由器

## 设置步骤

### 1. 进入网络设置

```
设置 → 网络 → 设定网络连接
```

### 2. 选择 Wi-Fi 并手动设定

选择你的 Wi-Fi 网络，在弹出的选项中不要直接连接，改为选右下角的「**手动设定**」。

如果 PS5 已经连上这个 Wi-Fi，按手柄 **Options 键** → 「设定」进入修改界面。

### 3. IP 地址设置

将「IP 地址设定」改为「**手动**」：

| 字段 | 填什么 | 说明 |
|------|--------|------|
| IP 地址 | `192.168.x.103` | 同网段未使用的 IP（x 和电脑相同） |
| 子网掩码 | `255.255.255.0` | 保持默认 |
| 默认网关 | `192.168.x.2` | **电脑的 IP** |

> 例：电脑 IP 是 `192.168.1.2`，PS5 可设为 `192.168.1.103`，默认网关填 `192.168.1.2`

### 4. DNS 设置

将「DNS 设定」改为「**手动**」：

| 字段 | 填什么 |
|------|--------|
| 主 DNS | 电脑的 IP（如 `192.168.1.2`） |
| 副 DNS | `8.8.8.8` |

### 5. 其他选项

| 字段 | 填什么 |
|------|--------|
| MTU 设定 | 自动 |
| 代理服务器 | **不使用** |

> **重要：** 代理服务器一定要选「不使用」。流量已经通过网关的 TUN 模式透明转发了，再配代理服务器会冲突导致无法上网。

### 6. 测试网络连接

完成设置后，选「**测试网络连接**」。

应该看到：
- 获取 IP 地址：成功
- 网络连接：成功
- PlayStation Network 登录：成功

## 关于 NAT 类型

通过网关代理后，PS5 的 NAT 类型通常显示为 **Type 2**（中等），偶尔 Type 3（严格）。

| NAT 类型 | 说明 | 影响 |
|----------|------|------|
| Type 1 | 直连，不经过网关 | 无法实现 |
| **Type 2** | 中等（通过网关） | 大部分游戏联机正常 |
| **Type 3** | 严格 | 部分游戏语音或 P2P 可能受限 |

大多数在线游戏在 Type 2 下完全正常。如遇 Type 3 且影响联机，在管理面板（`http://电脑IP:9090/ui`）切换代理节点尝试。

## 恢复自动网络

停止代理网关后，需要改回自动配置，否则会断网：

```
设置 → 网络 → 设定网络连接 → 选择你的 Wi-Fi
→ IP 地址设定：自动
→ DNS 设定：自动
→ 代理服务器：不使用
```

或者删除该 Wi-Fi 连接并重新连接（会自动恢复自动获取 IP）。

## 常见问题

**NAT 类型 Type 3，联机有问题**
这是通过网关代理的正常现象。在管理面板 `http://电脑IP:9090/ui` 切换不同代理节点尝试，部分节点 NAT 类型会更好。

**PSN 商店打不开 / 加载缓慢**
1. 运行 `gateway status` 确认网关正在运行
2. 通过管理面板切换到延迟更低的节点
3. 部分节点可能对 PSN 访问不佳，多换几个试试

**游戏下载速度慢**
下载速度取决于代理节点带宽。建议在管理面板测试延迟，选速度更快的节点。如果只是下载游戏不需要访问国外，可以临时把 PS5 网络改回自动（直连路由器），下载完再改回来。

**连不上 PlayStation Network**
1. 确认电脑网关正在运行（`gateway status`）
2. 确认 PS5 的默认网关和主 DNS 都填了电脑的 IP
3. 确认 IP 地址的前三段和电脑相同（如都是 `192.168.1.x`）
4. 确认代理服务器设置为「不使用」
